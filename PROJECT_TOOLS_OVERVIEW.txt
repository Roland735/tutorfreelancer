TutorFreelancer Project - Tools and Technology Overview
=======================================================

This document explains the major tools, frameworks, and libraries used in this project,
what each tool does, and how it is applied in TutorFreelancer.


1) Next.js (v16)
----------------
What it is:
- A React framework for building full-stack web applications.

Why it is used here:
- Provides file-based routing for pages and APIs.
- Supports server-rendered and client-rendered components.
- Simplifies deployment and production build workflow.

How it is used in this project:
- App routes in src/app/* for UI pages (login, signup, dashboard, jobs, tutors, etc.).
- API routes in src/app/api/* for backend endpoints (auth, profile, jobs, requests, stats, prediction).
- Script commands from package.json:
  - npm run dev
  - npm run build
  - npm run start


2) React (v19)
--------------
What it is:
- A JavaScript library for building reusable user interfaces with components.

Why it is used here:
- Helps build modular UI blocks and state-driven user experiences.

How it is used in this project:
- Component-driven UI in src/components/*.
- State and effects with hooks like useState and useEffect.
- Client components marked with "use client" where browser-side interaction is needed.


3) Tailwind CSS (v4) + @tailwindcss/postcss
--------------------------------------------
What it is:
- Utility-first CSS framework for rapidly building responsive UIs.
- PostCSS plugin integration for processing Tailwind directives.

Why it is used here:
- Speeds up styling with utility classes.
- Keeps styles consistent across pages and components.

How it is used in this project:
- Utility classes directly in JSX (spacing, color, typography, layout, responsiveness).
- UI components like buttons/cards/inputs use Tailwind class composition.
- Combined with tailwind-merge to safely merge class strings.


4) MongoDB + Mongoose (v9)
--------------------------
What it is:
- MongoDB: NoSQL database for storing platform data.
- Mongoose: ODM library for defining schemas/models and querying MongoDB.

Why it is used here:
- Flexible data model for users, jobs, sessions, requests, reviews, categories, etc.
- Schema validation and relationships (references between collections).

How it is used in this project:
- Database connector in src/lib/db.js using MONGODB_URI.
- Models in src/models/* (User, Job, Session, Request, Review, TutorProfile, etc.).
- API routes perform CRUD operations via Mongoose queries.
- Connection caching is implemented to avoid creating new DB connections on each request.


5) NextAuth.js (v4)
-------------------
What it is:
- Authentication solution for Next.js apps.

Why it is used here:
- Handles login sessions and secure user identity management.

How it is used in this project:
- Credentials provider in src/app/api/auth/[...nextauth]/route.js.
- Email/password auth flow with session/jwt callbacks.
- Custom sign-in page mapped to /login.
- Session data used in dashboards and protected API routes.


6) bcryptjs
-----------
What it is:
- Library for hashing and verifying passwords securely.

Why it is used here:
- Prevents storing plain text passwords.

How it is used in this project:
- Password comparison in auth route during login.
- Seed script uses hashed password values for generated users.


7) Anthropic SDK (@anthropic-ai/sdk)
------------------------------------
What it is:
- Official SDK for calling Anthropic Claude models.

Why it is used here:
- Used for AI-powered student prediction/insight generation.

How it is used in this project:
- Installed as a dependency and used in API logic for student prediction.
- Uses ANTHROPIC_API_KEY from environment variables.
- Claude model returns structured prediction output (risk level, engagement score, recommendations).
- If key/model call is unavailable, fallback logic can be used to keep feature available.


8) Faker (@faker-js/faker)
--------------------------
What it is:
- Fake data generation library for development/testing datasets.

Why it is used here:
- Quickly populates realistic sample records for development demos.

How it is used in this project:
- scripts/seed.js generates sample users, tutors, jobs, sessions, reviews, messages, and more.
- Helps test dashboards and listing pages with non-empty data.


9) Framer Motion
----------------
What it is:
- Animation library for React.

Why it is used here:
- Adds smooth transitions and interactive polish to the UI.

How it is used in this project:
- Used in animated sections/components where motion effects are needed.


10) Lucide React + React Icons
------------------------------
What they are:
- Icon libraries for React.

Why they are used here:
- Improves visual communication and UX clarity (actions, statuses, navigation cues).

How they are used in this project:
- Lucide icons are widely used across pages and cards.
- React Icons can be used for additional icon sets not in Lucide.


11) date-fns
------------
What it is:
- Utility library for date formatting/manipulation.

Why it is used here:
- Simplifies date/time formatting for sessions, jobs, and activity displays.

How it is used in this project:
- Used where date formatting logic is required in UI or API responses.


12) clsx + tailwind-merge
-------------------------
What they are:
- clsx: Conditionally builds className strings.
- tailwind-merge: Resolves conflicting Tailwind classes in merged strings.

Why they are used here:
- Keeps component styling logic clean and predictable.

How they are used in this project:
- Utility helpers combine static and dynamic classes for reusable UI components.


13) ESLint + eslint-config-next
-------------------------------
What it is:
- Static analysis tool for code quality and potential bug detection.

Why it is used here:
- Enforces consistent code standards.
- Catches common React/Next.js issues early.

How it is used in this project:
- Lint command: npm run lint
- Next.js lint rules enabled via eslint-config-next.


14) TypeScript (dev dependency support)
---------------------------------------
What it is:
- Typed superset of JavaScript.

Why it is present:
- Needed by some lint/tooling paths and future-proofing for typed code.

How it is used in this project:
- Currently project is mostly JavaScript, but TS tooling support is installed in dev dependencies.


15) dotenv
----------
What it is:
- Loads environment variables from .env files.

Why it is used here:
- Keeps secrets/config out of source code.

How it is used in this project:
- Used in scripts/seed.js to load .env.local.
- Environment-based values include:
  - MONGODB_URI
  - NEXTAUTH_SECRET
  - ANTHROPIC_API_KEY


16) Project Scripts and Workflow Tools
--------------------------------------
Core npm scripts:
- npm run dev: Starts local development server.
- npm run build: Creates production build.
- npm run start: Runs production server.
- npm run lint: Runs static lint checks.

Seeding workflow:
- scripts/seed.js populates development database with realistic records.
- Makes dashboards and data-driven pages testable immediately.


Summary
-------
The project is built as a full-stack Next.js application with:
- React for UI,
- Tailwind for design system and layout speed,
- MongoDB/Mongoose for data persistence,
- NextAuth + bcryptjs for authentication/security,
- Anthropic for AI-based student prediction features,
- and supporting tooling for icons, animation, linting, and data seeding.
